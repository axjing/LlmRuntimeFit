# LlmRuntimeFit

LLM运行时环境检查工具 - 自动检测、评估和优化大语言模型的运行环境

## 项目简介

LlmRuntimeFit 是一个功能强大的工具，用于检查和评估大语言模型（LLM）的运行环境。它能够：

- **自动检测模型信息**：无需手动配置，自动从模型路径获取参数量、序列长度等关键信息
- **智能硬件需求计算**：基于第一性原理，准确计算模型所需的GPU显存、CPU内存和核心数
- **硬件兼容性评估**：检测当前硬件状态，判断是否满足模型运行需求
- **性能测试**：支持vLLM和transformers两种引擎，测试模型的推理性能
- **自动优化配置**：根据模型大小自动选择最佳精度和量化策略

## 核心功能

### 1. 自动模型信息检测
- 从模型路径自动获取参数量、序列长度、隐藏层大小和层数
- 当自动检测失败时，使用合理的默认值作为fallback

### 2. 智能硬件需求计算
- 基于模型参数、精度和量化策略，准确计算所需硬件资源
- 考虑权重显存、KV缓存、激活值和框架开销
- 动态调整CPU核心需求

### 3. 硬件兼容性评估
- 检测当前硬件状态（GPU型号、显存、CPU核心数、内存）
- 判断硬件是否满足模型运行需求
- 提供详细的兼容性报告

### 4. 性能测试
- 支持vLLM（优先）和transformers两种推理引擎
- 测试首字时延（TTFT）、逐字时延（TPOT）和吞吐量
- 提供性能评估和优化建议

### 5. 自动优化配置
- 根据模型大小自动选择最佳精度（fp32/fp16）
- 大模型自动启用量化（int8）以减少内存需求
- 智能调整推理参数以获得最佳性能

## 使用方法

### 基本用法

只需指定模型路径，其他参数会自动检测和设置：

```python
from check_llm_runtime_environment_optimized import LLMRuntimeChecker

# 配置参数
config_dict = {
    "model_path": "Qwen/Qwen3.5-4B"
}

# 创建检查器并运行测试
checker = LLMRuntimeChecker(config_dict)
checker.run_full_test()
```

### 高级配置

如果需要自定义配置参数，可以在config_dict中指定：

```python
config_dict = {
    "model_path": "Qwen/Qwen3.5-4B",
    "precision": "fp16",  # 精度：auto/fp32/fp16/bf16/int8/int4
    "quantization": "int4",  # 量化：auto/fp32/fp16/bf16/int8/int4
    "batch_size": 1,  # 批次大小
    "seq_len": 8192,  # 序列长度
    "max_new_tokens": 512,  # 最大生成token数
    "device": "auto",  # 设备：auto/cuda/cpu
    "engine": "auto"  # 引擎：auto/vllm/transformers
}
```

### 命令行运行

直接运行脚本：

```bash
python check_llm_runtime_environment_optimized.py
```

默认会检查 `Qwen/Qwen3.5-4B` 模型的运行环境。

## 支持的模型

该工具支持大多数基于transformers的大语言模型，包括但不限于：

- Qwen系列（Qwen3.5-4B, Qwen3.5-7B等）
- Llama系列（Llama 2, Llama 3等）
- Mistral系列
- Gemma系列
- 其他Hugging Face上的模型

## 注意事项

1. **网络连接**：首次运行时需要下载模型配置和权重，确保网络连接正常
2. **内存需求**：运行较大模型时，确保有足够的CPU内存用于模型加载
3. **vLLM依赖**：vLLM需要CUDA 11.7+和足够的GPU显存
4. **自动检测**：自动检测可能会因模型结构不同而有所差异，如有必要可手动指定参数

## 故障排除

### 常见问题

1. **自动检测失败**：如果自动检测模型信息失败，会使用默认值，可手动指定相关参数
2. **vLLM加载失败**：如果vLLM加载失败，会自动回退到transformers引擎
3. **内存不足**：如果硬件不满足需求，会提供详细的原因说明

### 解决方法

- **模型过大**：尝试使用更小的模型或启用量化
- **显存不足**：减少序列长度或批次大小
- **CPU内存不足**：关闭其他占用内存的程序
